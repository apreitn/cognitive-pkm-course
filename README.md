# Cognitive PKM Course Repository

## Purpose
This repository is my personal knowledge management environment for the Software Engineering course.

The goal is not only to collect notes, but to document thinking, decisions, experiments, and learning progress using Git.

## Main principles
- Notes should be structured and easy to review.
- Important decisions should include the reason behind them.
- Experiments should describe what was tested and what was learned.
- Learning should be versioned through Git commits.
- The repository should show progress during the semester.

## Folder structure
- `lectures/` — lecture notes and summaries
- `insights/` — important ideas and lessons learned
- `decisions/` — decisions made during the course
- `experiments/` — small tests, prototypes, and technical experiments
- `resources/` — useful links, references, and materials

## Rules for using this repository
1. Add notes regularly after lectures or practical work.
2. Use clear file names.
3. Keep notes short but meaningful.
4. Write down why decisions were made, not only what was decided.
5. Use Git commits to track the development of my understanding.
6. Keep this repository public during the course.
